About us


We implement a system which is able to generate website usage metrices in json format and monitor it to perform necessary updates on deployment yaml and dockerfiles. The goal is to monitor the RAM and CPU usage of kubernetes pod and docker container. We are generating fake usage metrices in our project. In order to get real metrices we need a real website having very huge number of users per hour (not feasible for us). .Out project consist of 2 modules

1-Github project cloniong and deployment

In this module system clone (download) github project url provided by us and then create a docker image of it and push that docker image to docker hub. At the end system create 2 files for deployment purpose one is services.yaml and other is deployment.yaml, both files can be use to deploy our cloned project and then project will accessible from localhost

2-Generating data and monitoring

In this module system will generate data.json and config.json files. Monitor the data.json file and if system found that metrices exceeding threshoild values then generate new services.yaml, deployment.yaml and dockerfile according to the requirements define in the config.json file

What is the purpose of new generated files ?

You will get 3 types of files for deployment

  • Dockerfile
  • services.yaml
  • deployment.yaml

Dockerfile

This file define how many containers you need to create. It's upto you how you want to create container from same dockerImage from different dockerImage

deployment.yaml

This file include the defination of our docker image. Every file define only one docker image

services.yaml

This define how many services (endoints) we need for a specific deploymnent.It can be localhost:3000

How to use new generated file ?

Procedure include following steps

  • Create a dockerImage from dockerFile.
    • docker build -t yourDockerHubUsername/appName:tagName
  • push dockerImage to your docker hub account
    • docker push dockerImageName
  • Write the name of your pushed docker image in deployment.yaml file
  • Deploy the deployment.yaml and services.yaml on kubernetes by these commands
    • kubectl apply -f deployment.yaml
    • kubectl apply -f services.yaml

Future work

Usage metrices of real website can be get by using prometheus or any other latest tool and then monitoring can be apply on those real metrices. Also can work on complete data driven website and using volumes and databases in dockerfiles
